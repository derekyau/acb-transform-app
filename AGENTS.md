# AGENTS.md

## Project Overview

ACB Transform is an Electron desktop app built with React, TypeScript, and Vite. It provides a desktop UI for selecting broker CSV files, choosing a transformer, and writing an output CSV.

The Electron renderer must not receive direct Node.js access. Keep file system work, dialogs, and transform execution in the main process and expose only narrow methods through the preload bridge.

## Runtime

Use Node.js 24 or newer.

```bash
nvm use
npm install
```

The repo includes `.nvmrc` with the currently used Node 24 version.

## Development Mode

Run:

```bash
npm run dev
```

This starts three processes:

- Vite renderer dev server on `http://127.0.0.1:5173`
- TypeScript watch compilation for Electron main and preload code
- Electron pointed at the Vite dev server

If the Electron app does not open on the first run, check that `dist/main/main.js` and `dist/main/preload.js` were generated by the watch compiler.

## Build and Release

Run a production build:

```bash
npm run build
```

This type-checks the renderer and Electron code, builds the Vite renderer to `dist/renderer`, and compiles Electron main/preload/transformer files into `dist/main`.

Build macOS release artifacts:

```bash
npm run dist
```

Release output is written to `release/`. The current electron-builder config creates macOS DMG and ZIP artifacts.

## Key Files

- `src/main/main.ts`: Electron app lifecycle, window creation, and IPC handlers.
- `src/main/preload.ts`: secure context bridge exposed to the renderer.
- `src/shared/ipc.ts`: shared IPC request/result types.
- `src/renderer/App.tsx`: React UI for selecting files and running transforms.
- `src/main/transformers/run-transform.ts`: validation and transformer dispatch.
- `src/main/transformers/acb-ibkr-transform.js`: IBKR transformer stub.
- `src/main/transformers/acb-qt-transform.js`: QT transformer stub.

## Transformer Work

Real transformation logic belongs in the two transformer files:

- `src/main/transformers/acb-ibkr-transform.js`
- `src/main/transformers/acb-qt-transform.js`

Each transformer should keep exporting:

```js
async function transform({ inputPath, outputPath }) {
  // implementation goes here
}
```

Keep broker-specific parsing and mapping isolated in those files or nearby helper modules under `src/main/transformers`.

## Verification

```bash
npm run lint
npm run build
```

For packaging changes, also run:

```bash
npm run dist
```
